# 📅 2026-04-24 Oracle_Tuning
### 🏷️ Topic
SQL JDBC와 튜닝

### 📝 What I Learned
SQL 튜닝을 하면서 그동안 배운 내용을 바탕으로 쿼리문을 작성 한 후 튜닝을 하며 교정했다.
먼저 데이터베이스에서 가장 중요하는 것 =>  속도 최적화 => INDEX  
INDEX : 

        데이터 양이 많을 때 (보통 수만 건 이상) 사용

        WHERE 절에 자주 등장하는 컬럼에 사용

        중복된 값이 적은 컬럼에 사용 => 선택도가 좋다

        JOIN, ORDER BY에 자주 사용되는 컬럼에 사용 => 인덱스가 있으면 미리 정렬된 데이터를 쓰기 때문에 빠르다

INDEX 생성 : 

            CREATE INDEX 인덱스명 ON 테이블명(컬럼명);

인덱스 단점 : 

            인덱스가 많으면 조회(SELECT)는 빠르지만 수정(INSERT, UPDATE, DELETE)는 느려진다.
            내용이 수정 될 때마다 순서대로 다시 정리해야하기 때문 

### 💡 Note
영화 목록, 상세보기, 검색을 강사님과 함께 만들어 보았는데 JDBC를 중심으로 DAO, VO쪽에 집중하며 윈도우창을 만들었다

이 후 혼자서 영화와 같은 양식으로 음악 목록, 상세보기, 검색 윈도우 창을 직접 만들어 보았다.

DAO 작성 순서 : 
    
                  드라이버 등록: Class.forName()으로 오라클 드라이버 준비
                  
                  오라클 연결: DriverManager.getConnection(URL, id, pw)로 접속
                  
                  SQL 문장 제작: String sql 작성 (SELECT/INSERT/UPDATE/DELETE)
                  
                  SQL 전송 준비: conn.prepareStatement(sql)
                  
                  실행 및 결과 받기: executeQuery() (SELECT) 또는 executeUpdate() (DML)
                  
                  데이터 채우기: 결과를 List나 VO 객체에 매핑
                  
                  자원 닫기: ps.close(), conn.close() (필수!)
