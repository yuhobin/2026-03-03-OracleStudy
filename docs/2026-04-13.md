# 📅 2026-04-10 Oracle_SELECT

### 🏷️ Topic
데이터베이스의 기본 개념과 SQL언어의 기본 규칙 및 SELECT절

### 📝 What I Learned
데이터베이스의 기본 개념과 특징을 알아보고 SQL언어의 구성을 알아본 후 그 중 SELECT절에 대해 구조를 중심으로 알아보았다.

데이터베이스는 4가지 특징이 있는데 

1. 실시간 접근성

2. 계속적인 변화

3. 동시 공유

4. 내용에 의한 참조

키의 종류 : ***기본키, 후보키, 대체키, ***외래키, 슈퍼키

데이터베이스 언어는 사용 목적에 따라 DDL, DML, DCL, TCL 로 나눠진다.

DDL(데이터 정의어) : ALTER CREATE, DROP

DML(데이터 조작어) : SELECT, INSERT, DELETE, UPDATE

DCL(데이터 제어어, 권한 관련) : GRANT, REVOKE

TCL(저장 관련) : COMMIT, ROLLBACK 

SELECT는 기본적으로 

----------------------------------------

SELECT * (ALL)|원하는 컬럼지정 column_list

FROM table_name

---------------------------------------- 여기까진 필수

WHERE 조건(연산자)

GROUP BY 그룹 컬럼

HAVING 그룹 조건 ==> (GROUP BY가 있을 경우만 사용 가능)

ORDER BY 정렬 컬럼 (ASC|DESC)

-----------------------------------------


이렇게 구성되었다.

SQL은 주의 할 점이

1. 대소문자 구분이 없다 Select / select / SELECT 모두 가능하다 (단, ** 키워드는 대문자로 쓰는것이 약속이다)

2. 실제 저장된 데이터는 대소문자를 구분한다 (KING => king (X))

3. 문장이 종료되면 ;를 사용한다. (단, 자바에서는 자동으로 ; 이 추가된다.)

4. 문자, 날씨는 반드시 ' ' 를 사용한다.
