# 📅 2026-04-16 Oracle_JOIN_SUBQUERY
### 🏷️ Topic
오라클 JOIN과 SUBQUERY

### 📝 What I Learned
JOIN 과 SUBQUERY에 대해 학습했다. 

JOIN :

INNER JOIN :  두개 이상의 테이블 에서 공통으로 존재하는 값을 이용해서 데이터만 조회 (교집합)

			문법 형식
            SELECT 출력하는 데이터 
			FROM table1, table2
			WHERE table1.col=table2.col => 별칭 사용이 가능
                   
			EQUI_JOIN : 같은 값인 경우
      		
			NON_EQUI_JOIN : 범위에 포함된 경우 ( BETWEEN, 비교, 논리)

			SELF JOIN : 같은 테이블로 조인

* 단점은 null 인 경우에는 처리하지 못한다. 그래서 OUTER JOIN 등장

OUTER JOIN :  한쪽 테이블 기준으로 모든 데이터를 출력 =NULL인 경우에는 출력이 안된다    

            = LEFT OUTER JOIN
			INTERSECT + MINUS A-B
                        
			= RIGHT PUTER JOIN
			INTERSECT + MINUS B-A
     
			= FULL OUTER JOIN
			UNION ALL

			문법 형식

			ANSI 조인
			SELECT A.col, B.col
			FROM A LEFT OUTER JOIN B
			ON A.col=B.col
				
		=>	오라클 조인
			SELECT A.col, B.col
			FROM A ,B
			WHERE A.col=B.col(+)

SUBQUERY : SQL 문장 여려개를 연결. 쿼리를 합친다, => 웹 개발자들이 자주 사용(네트워크 통신), DML 전체에서 사용이 가능
			
			종류
			1. WHERE 뒤에 일반 서브쿼리  -------------------> 조건문
			
				MainQuery
				WHERE (SubQuery)
					   -------------1 => 결과값을 MainQuery로 전송 => 결과값 출력
				SQL + SQL = 서브쿼리
				데이터 + 데이터 = JOIN 
			
			2. SELECT 뒤에 스칼라 서브쿼리*** (JOIN 대체) ---> 칼럼
			3. FROM 뒤에 인라인뷰*** -----------------------> 테이블
				
				인라인뷰 / 스칼라 서브쿼리
				FROM(SELECT~) => 보안
				SELECT (SELECT~) => 조인 대신

EXISTS와 IN 
			
			IN => 값 비교
			EXISTS => 존재여부만 확인 => 속도가 빠르다
