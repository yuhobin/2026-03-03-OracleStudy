# 📅 2026-04-27 Oracle_ER-Moeling
### 🏷️ Topic
DB ER-Modeling 및 상품목록 만들기 및 회원가입

### 📝 What I Learned
데이터 모델링의 3단계 : 개념적 모델링 -> 논리적 모델링 -> 물리적 모델링

개념적 모델링 : 
                목적 : 업무의 핵심 개념 구분 및 전체 뼈대 구성
                주요 작업 : 개체 추출, 관계 정의
                산출물 : ER 다이어그램

논리적 모델링 : 

                목적 : 목표 DBMS에 맞춘 매핑
                주요 작업 : 상세속성 추출, 정규화, 데이터 표준화
                산출물 : 관계 스키마 모델

물리적 모델링 : 
                목적 : 실제 저장 장치에 맞춘 물리적 구조 정의
                고려사항 : 응답시간 최소화, 동시 트랜잭션 수용, 저장공간 효율성
                산출물 : 물리적 데이터베이스 스키마

ER 모델 : 사각형(개체), 마름모(관계), 타원(속성)
  
          사각형(개체) : 테이블
          타원 (속성) : 컬럼 - 개체가 가진 고유한 성질
          마름모 (관계)  : 개체 사이의 연관성
                      

### 💡 Note
배운 ER 모델링을 프로젝트에 적용 시키기 위해 요구사항 분석으로 해보며 설계 과정 진행

도서관 관리 프로그램 관계 : 

                      도서관 => 회원 / 도서 / 대출 / 구매 
                                 |        |  -------------
                                  ---------    | 관계 
                                  | 테이블
                      회원 : 회원번호 / 이름 / 전화번호 
                      도서 : ISBN , 제목 , 저자 
                      대출 : 대출일 / 반납일 
                      구매 : 구매일 / 수량 / 도서명 / 회원 번호 

                      회원 - <대출> - 도서 
                      회원 - <구매> - 도서  => N:M

도서관리 프로그램 설계 : 
                       => 키오스크 도서 , 대여 , 회원 관리를 한다 
                       => 회원은 키오스크를 이용해서 책 대여 / 반납이 가능 
                       => 한명의 회원은 여러 책를 대여할 수 있다 
                       => 대여시엔, 대여일 , 반납예정일 , 실제 반납일 
                       => 도서 ISBN 제목  카테고리 , 이미지 
                       => 회원 ID , PWD 이름 , 전화번호 ...
                       => 회원 = <대여> = 도서 : N:M
