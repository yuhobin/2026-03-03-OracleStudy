### 🏷️ Topic
WHERE, ORDER BY, 연산자, 집계함수, JDBC 연동하기

### 📝 What I Learned
WHERE 절 이후에 들어올 조건을 작성할때 들어올 연산자와 집계함수를 학습했다.

산술 연산자 : 

			 (+, -, *, /)를 사용하는데 / 사용시 나머지는 표기하지 않는다. 당연히 0으로 나눌 수 X

비교 연산자 : 

			 =(같다), !=(같지않다), <>(권장),^=

             < (작다), > (크다), <= (작거나 같다), >= (크거나 같다)

논리 연산자 : 

		     AND => &&를 사용하면 안된다 (&는 입력값을 받는 경우에 사용)

             OR => || (문자열 결합에 사용)
	
             NOT = !을 사용하면 안된다
		
             NOT IN / NOT BETWEEN / NOT LIKE
    
             AND / OR

IN => OR 여러개 대체

IS NULL , IS NOT NULL ==> null 값은 연산처리 시  결과값은 0이 아닌 null이다

 LIKE => SELECT 문장의 핵심은 검색 => 패턴
		
		% : 문자의 개수를 모르는 경우
		
		_ : 문자의 한글자

ORDER BY : ORDER BY 컬럼 (ASC|DESC), 마지막에 배치
		


내장함수 중 집계함수

		SUM : column의 총합 => 장바구니 / 구매 금액
		
		AVG : column의 평균 => 전체 구매자의 평균 금액 => 통계
		
		MIN : column의 최솟값
		
		MAX : column의 최댓값 ====> 자동 증가번호(중복이 없이)
		
		*** COUNT : column의 개수 : 로그인 / 아이디 중복 체크 / 검색 결과
		
		RANK() : 순위 출력, 1 2 2 4
		
		DENSE_RANK() : 순위 출력 1 2 2 3
