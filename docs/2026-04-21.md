# 📅 2026-04-21 Oracle_Index
### 🏷️ Topic
SQL의 속도개선 INDEX

### 📝 What I Learned
SQL에서 속도를 개선하는 방법 INDEX / PL/SQL / SQL튜닝 이 있다.

INDEX : 

		검색 (빠른속도)=> 대용량 유리
		
        장점 : 검색 속도향상
		
		단점 : 저장공간이 많이 필요
			=> 서버 부하를 줄이기 위해 => 한테이블에 (4~5 정도 적당)
			=> 맛집 => 주소 / 맛집명 / 음식종류
			=> 맛집 <=> 레시피 => JOIN컬럼
			=> 테이블 데이터가 자주 변경시에는 속도가 느려진다 
				=> 회원가입, 게시판, 묻고 답하기, 댓글, 공지사항 => PK
				=> ALTER INDEX index명 REBUILD
				=> DML이 많은 경우에는 인덱스가 속도 저하

		** 데이터베이스에서 데이터 빨리 찾기 위해 사용되는 구조
      	** 인덱스가 없는 경우 : Full Scan => 전체를 검색


INDEX 생성시기

              구별되는 값을 가지고 있는 경우 (PRIMARY KEY)
			  ROW가 많은 경우(10000개)
              WHERE절에서 자주 검색되는 컬럼이 있는 경우
              JOIN에서 주로 사용되는 컬럼 (deptno, 가수명, 곡명) 
              NULL값을 포함하고 있는 컬럼 (comm,bunji)

INDEX 사용 
			생성
				CREATE INDEX index명 ON 테이블 (컬럼 (ASC|DESC))
				CREATE INDEX index명 ON 테이블 (컬럼 (ASC|DESC), 컬럼 (ASC|DESC))

			삭제
				DROP INDEX index명
			리빌드
				ALTER INDEX index명 REBUILD
			정렬
				--+
				/*+ */ => INDEX(테이블 인덱스명)
						INDEX_ASC (테이블 PK)
						INDEX_DESC (테이블 PK)
		------------------------------------------------------------

INDEX 속도가 저하되는 상태 

                          INSERT, UPDATE, DELETE가 많은 경우 (게시판, 댓글...)
                          데이터가 자주 변경되면 => Index를 다시 설정 (REBUILD)

                         
INDEX 활용
			  ==> 인덱스 적용 
			   1) ORDER BY 컬럼 = >인덱스 컬럼 순서와 동일
		       2) ASC / DESC 방향 일치
			   3) WHERE 조건 => 인덱스와 일치
						  
		       ==> 인덱스 적용이 안됨
				ORDER BY score+1 => 연산처리
				ORDER BY name||type => 가공
						 
				==> 인덱스를 필요로 하는 곳
				1) WHERE 조건 검색
			    SELECT *
			    FROM food 
				WHERE name='' => name을 인덱스로 속도 최적화
							
				2) JOIN 
				SELECT *
			    FROM  emp
			    JOIN dept ON emp.deptno=dept.deptno
			    => deptno 인덱스
							
			    3) ORDER BY
			    SELECT *
			    FROM food
		        ORDER BY score DESC ==> ON food(score DESC)
							
		 		4) GROUP BY
				SELECT type,count(*)
				FROM food
				GROUP BY type ==> type => INDEX
