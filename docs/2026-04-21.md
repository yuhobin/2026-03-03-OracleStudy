# 📅 2026-04-21 Oracle_Index
### 🏷️ Topic
SQL의 속도개선 INDEX

### 📝 What I Learned
SQL에서 속도를 개선하는 방법 INDEX / PL/SQL / SQL튜닝 이 있다.

INDEX : 
        ** 데이터베이스에서 데이터 빨리 찾기 위해 사용되는 구조
      	** 인덱스가 없는 경우 : Full Scan => 전체를 검색
        검색시에 속도 향상
        인덱스를 저장하기 위해 메모리가 많이 필요 => 인덱스를 많이 생성하면 속도가 느려지니 사용후 삭제필요

INDEX 생성시기

              구별되는 값을 가지고 있는 경우 (PRIMARY KEY)
              WHERE절에서 자주 검색되는 컬럼이 있는 경우
              JOIN에서 주로 사용되는 컬럼 (deptno, 가수명, 곡명) 
              NULL값을 포함하고 있는 컬럼 (comm,bunji)

INDEX 속도가 저하되는 상태 

                          INSERT, UPDATE, DELETE가 많은 경우 (게시판, 댓글...)
                          데이터가 자주 변경되면 => Index를 다시 설정 (REBUILD)
