### 📝 What I Learned
PL/SQL : 프로시저를 만드는 언어  (기능 = 함수 = 재사용이 가능)

    => 프로그램은 SQL문장으로 제작 : 명령어를 모아서 처리
	  => 속도가 빠르다 / 절대 지워지지 않는다, 보안 
	  => 단점 : 웹 개발자가 수정하기 어렵다 , 가독성이 떨어진다
        
  	=> FUNCTION : 리턴형이 있는 경우 (처리 결과값)
  	=> PROCEDURE : 리턴형이 없는 경우 (void)
  	=> TRIGGER : 자동화 처리 => 자바자체에 소스 코딩 

형식 : 

      DECLARE
				=> 선언부 : 변수 선언 
			BEGIN
				=> 구현부 : SQL문장으로 주로 사용
			END;
			/
      
변수 종류 : 

          = 일반 변수 (스칼라 변수)
					  변수 데이터형(VARCHAR2, DATE, NUMBER, CLOB)
				  	----------------
				  	empno NUMBER(4)
				  = %TYPE : 실제 테이블에 컬럼의 데이터형을 가지고 온다
					  변수 테이블명.컬럼명%TYPE => 가장 많이 사용되는 데이터형
						------------------------- 한개씩
			  	= %ROWTYPE : 테이블에 있는 컬럼의 모든 데이터형을 읽는다
				  	변수 테이블명%ROWTYPE ==> 자바 (VO)
					=> 한개의 테이블만 가능 
			    	= RECORD : 테이블 여러개 연결 => JOIN, SUBQUERY => 사용자 정의 데이터형   ------- 여기까진  ROW 한개의 데이터만 받는다

				  = CURSOR  : ROW 여러개를 동시에 저장 => 자바로 전송
					  => ResultSet / ArrayList
